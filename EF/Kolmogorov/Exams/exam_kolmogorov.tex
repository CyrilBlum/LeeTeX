	\question[2] Wir betrachten die Wörter $w_n = 0^n1^{n^2}$ für alle $n\in\N$. Welches der folgenden drei Programme $A_n, B_n, C_n$ liefert asymptotisch die beste obere Schranke für die Kolmogorov-Komplexität von $w_n$? Begründe deine Antwort.

	\textbf{Bemerkung:} Der Ausdruck \verb|n**2| in \cref{listing:Cn} ist als explizite binäre Darstellung der Zahl $n^2$ zu verstehen und trägt $\abs{\text{Bin}\lr{n^2}}$ viele Bits zur binären Länge von $C_n$ bei.
\begin{lstlisting}[language=C++,caption=$A_n$]
    #include <iostream>
    #include <string>
    
    void An() {
        int x = ***n+++;
        int y = 0;
        for (int i = 1; i <= x; ++i) {
            std::cout << 0;
            y = y + x;
        }
        for (int j = 1; j <= y; ++j) {
            std::cout << 1;
        }
        std::cout << std::endl;
    }
\end{lstlisting}

\begin{lstlisting}[language=C++,caption=$B_n$]
    #include <iostream>
    #include <string>
    
    void Bn() {
        for (int i = 1; i <= ***n+++; ++i) {
            std::cout << 0;
        }
        for (int i = 1; i <= ***n+++; ++i) {
            for (int j = 1; j <= ***n+++; ++j) {
                std::cout << 1;
            }
        }
        std::cout << std::endl;
    }
\end{lstlisting}

\begin{lstlisting}[language=C++,caption=$C_n$,label=listing:Cn]
    #include <iostream>
    #include <string>
    
    void Cn() {
        for (int i = 1; i <= ***n+++; ++i) {
            std::cout << 0;
        }
        for (int i = 1; i <= ***n**2+++; ++i) { // n**2 entspricht n*n in C++ 
            std::cout << 1;
        }
        std::cout << std::endl;
    }
\end{lstlisting}
	\begin{solution}
		Es lässt sich leicht nachprüfen, dass alle drei Programme jeweils die Ausgabe $w_n$ generieren. Die oberen Schranken für $K(w_n)$ ergeben sich aus der Länge der Darstellung der jeweiligen Programme in Maschinencode. Die Programme haben alle eine Konstante Länge, bis auf die Teile, die vom Parameter $n$ abhängen. Die Darstellung der Zahl $n$ benötigt eine Länge von $\abs{\text{Bin}\lr{n}}$ Bits. Die beste obere Schranke liefert Programm $A_n$, da der Parameter $n$ nur einmal vorkommt. Die Länge des Maschinencodes von $A$ ist $\ceil{\log_2(n+1)} + c_A$ und somit liefert $A_n$ die obere Schranke
		\begin{align*}
			K(w_n) \leq \ceil{\log_2(n+1)} + c_A.
		\end{align*}
		Analog liefert $B_n$ die Schranke
		\begin{align*}
			K(w_n) \leq 3\ceil{\log_2(n+1)} + c_B
		\end{align*}
		und $C_n$
		\begin{align*}
			K(w_n) \leq \ceil{\log_2(n+1)} + \ceil{\log_2\lr{n^2+1}} + c_C.
		\end{align*}
	\end{solution}


	\question[2] Sei $w_n := 0^{2^{2^{2n^4}}}$ für jedes $n\in\N, n>0$ ein binäres Wort. Bitte beachte unbedingt, dass Potenzausdrücke der Form $b^{p^q}$ gelesen werden müssen als
	\begin{align*}
		b^{p^q} := b^{\lr{p^q}}, \quad \text{also zum Beispiel} \quad 2^{3^2} := 2^{\lr{3^2}} = 2^9 = 512 \neq \lr{2^3}^2 = 8^2 = 64.
	\end{align*}
	Somit gilt
	\begin{align*}
		w_n := 0^{2^{2^{2n^4}}} := 0^{2^{\lr{2^{\lr{2n^4}}}}}.
	\end{align*}
	Gib eine (asymptotisch) möglichst gute obere Schranke für die Kolmogorov-Komplexität $K(w_n)$ von $w_n$ an, gemessen in der Länge $\abs{w_n}$ von $w_n$.
	\begin{solution}
		Wenn wir die Operation \verb|^| für die Exponentiation zulassen (dies ist nicht ursprüngliche Pascal-Syntax, wir wollen diese Notation aber im Folgenden verwenden, um die
		Programme lesbarer darzustellen), erzeugt auch das folgende Programm das Wort $w_n$:
		\begin{lstlisting}[language=C++,caption=$W_n$]
    #include <iostream>
    #include <cmath> // für die pow Funktion
    
    void Wn() {
        int k = ***n+++;
        int k = 2 * pow(k, 4);
        int k = pow(2, pow(2, k));
        for (int i = 1; i <= k; ++i) {
            std::cout << 0;
        }
        std::cout << std::endl;
    }
\end{lstlisting}
		Die Maschinencodes dieser Programme $W_n$ sind alle identisch und unabhängig von $w_n$ (und haben somit eine konstante binäre Länge) mit Ausnahme der Darstellung von $n$ in der Zeile Nummer 2. Somit ist die binäre Länge eines Programms $W_n$ gegeben durch
		\begin{align*}
			\ceil{\log_2(n+1)} + c
		\end{align*}
		für eine Konstante $c$. Damit lässt sich die Kolmogorov-Komplexität von $w_n$ abschätzen durch
		\begin{align*}
			K(w_n) \leq \ceil{\log_2(n+1)} + c \leq \blue{\log_2(n)} + c'
		\end{align*}
		für eine Konstante $c' := c + 1$. Die (binäre) Länge von $w_n$ ist
		\begin{align*}
			\abs{w_n} = 2^{2^{2n^4}}.
		\end{align*}
		Nach $n$ auflösen liefert
		\begin{align*}
			n = \lr{\frac{\log_2\lr{\log_2\lr{\abs{w_n}}}}{2}}^{1/4}
		\end{align*}
		und somit
		\begin{align*}
			\blue{\log_2(n)} & = \log_2\lr{\lr{\frac{\log_2\lr{\log_2\lr{\abs{w_n}}}}{2}}^{1/4}} = \\
			                 & = \frac{1}{4}\lr{\log_2\lr{\log_2\lr{\log_2\lr{\abs{w_n}}}}-1}.
		\end{align*}
		Dies ergibt die obere Schranke
		\begin{align*}
			K(w_n) \leq \frac{1}{4}\log_2\lr{\log_2\lr{\log_2\lr{\abs{w_n}}}} + c''
		\end{align*}
		für die Kolmogorov-Komplexität von $w_n$, wobei $c'':=c'-1/4$ eine Konstante ist.
	\end{solution}

	\question[1] Zeige, dass es für jede natürliche Zahl $n$ und jedes natürliche $i$ mit $i<n$ in dem Intervall $\ic{2^n}{2^{n+1} - 1}$ mindestens $2^n - 2^{n-i}$ natürliche Zahlen $x$ gibt mit $K(x) \geq n-i$.
	\begin{solution}
		Das Intervall $\ic{2^n}{2^{n+1} - 1}$ enthält genau $2^n$ natürliche Zahlen mit einer kürzesten binären Darstellungslänge von $n + 1$.
		Da die binären Darstellungen aller dieser Zahlen mit einer $1$ beginnen (die führende $1$ ist bereits gesetzt), können wir diese Zahlen identifizieren mit den binären Wörtern der Länge $n$.

		Die Anzahl der verschiedenen nichtleeren Maschinencodes der Länge kleiner als $n - i$ ist höchstens
		\begin{align*}
			\sum_{k=1}^{n - i - 1} 2^k = 2^{n-i} - 2,
		\end{align*}
		da es für jede Länge nicht mehr Maschinencodes als nichtleere Binärstrings geben kann. Damit existieren maximal $2^{n-i} - 2$ Wörter $w$ mit $K(w) < n - i$. Für die Anzahl $m$ der binären Wörter $w$ der Länge $n$ mit $K(w)\geq n - i$ gilt also
		\begin{align*}
			m\geq 2^n - (2^{n-1} - 2) > 2^n - 2^{n-i}.
		\end{align*}
	\end{solution}
