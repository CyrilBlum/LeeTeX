\begin{cheatSheetBox}[title=Spalten auswählen: \texorpdfstring{\lstinline|SELECT|}{SELECT}]
Auswahl aller Spalten der Tabelle:
\begin{lstlisting}
SELECT *
FROM users
\end{lstlisting}

Auswahl der Spalten \texttt{city} und \texttt{gender} der Tabelle \texttt{users}:
\begin{lstlisting}
SELECT city, gender
FROM users
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Duplikate löschen: \texorpdfstring{\lstinline|DISTINCT|}{DISTINCT}]
Mit dem Zusatz \lstinline|DISTINCT| werden Duplikate aus den Resultaten gelöscht.
\begin{lstlisting}
SELECT DISTINCT gender
FROM users
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Sortieren: \texorpdfstring{\lstinline|ORDER BY|}{ORDER BY}]
Mit dem Befehl \lstinline|ORDER BY| können Resultate sortiert werden, nach einer oder mehreren Spalten. Mit folgendem Befehl erhalten wir die Namen aller Users, sortiert nach Stadt und nach Region:
\begin{lstlisting}
SELECT name, city
FROM users 
ORDER BY city ASC, name ASC
\end{lstlisting}
\begin{itemize}
\item \lstinline|DESC| = absteigend (\textit{descending})
\item \lstinline|ASC| = aufsteigend (\textit{ascending})
\end{itemize}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Erste / Letzte Zeilen: \texorpdfstring{\lstinline|LIMIT|}{LIMIT}]
Mit dem Befehl \lstinline|LIMIT| können eine Tabelle auf die ersten \textit{n} Zeilen beschränkt werden. Z.B. können die Namen der ersten drei Personen der Tabelle \texttt{users} wie folgt abgefragt werden:
\begin{lstlisting}
SELECT name
FROM users
LIMIT 3
\end{lstlisting}
\end{cheatSheetBox}

%\subsection*{Filtern}
\begin{cheatSheetBox}[title=Zeilen Filtern: \texorpdfstring{\lstinline|WHERE|}{WHERE}]
Mit dem Befehl \lstinline|WHERE| können die Resultate einer Abfrage nach eigenen Kriterien gefiltert werden:
\begin{lstlisting}
SELECT name, city, gender
FROM users
WHERE gender='male'
\end{lstlisting}

\begin{table}[H]
  \centering
  \centering\resizebox{\columnwidth}{!}{
  \begin{tabular}{c|l}
    \textbf{Operatoren} & \textbf{Bedeutung} \\\midrule
    \texttt{=}                                    & gleich ($=$)                                \\
    \texttt{<>}                                   & ungleich ($\neq$)                           \\
    \texttt{<}                                    & kleiner als ($<$)                           \\
    \texttt{<=}                                   & kleiner oder gleich ($\leq$)                \\
    \texttt{>}                                    & grösser als ($>$)                           \\
    \texttt{>=}                                   & grösser oder gleich ($\geq$)                \\
    \lstinline|BETWEEN x AND y|        & ein Wert zwischen \texttt{x} und \texttt{y} \\
    \lstinline|IN (wert1, wert2, ...)| & einer von mehreren Werten         \\
    \lstinline|IS NULL|                & Wert ist leer
  \end{tabular}}
\end{table}
\end{cheatSheetBox}
\begin{cheatSheetBox}[title=Filter kombinieren: \texorpdfstring{\lstinline|AND|, \lstinline|OR|}{AND, OR}]
Mehrere Filter können mit \lstinline|AND| (\enquote{und} $\rightarrow$ beides muss wahr sein) oder \lstinline|OR| (\enquote{oder} $\rightarrow$ eine der beiden Bedingungen muss wahr sein) verbunden werden.

Users aus Leipzig, Berlin oder Hamburg, welche grösser als 170 sind:
\begin{lstlisting}
SELECT name, city
FROM users 
WHERE city IN ('Leipzig','Berlin','Hamburg')
 AND centimeters > 170
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Ungefähre Treffer: \texorpdfstring{\lstinline|LIKE|}{LIKE}]
Mit folgendem Befehl erhalten wir alle Städte, die mit ``Be...'' beginnen:
\begin{lstlisting}
SELECT DISTINCT city 
FROM users 
WHERE city LIKE 'Be%'
\end{lstlisting}
Das Prozentzeichen (\%) ist ein sogenannter \textit{Platzhalter} und steht für \enquote{irgendetwas kommt hier (oder nicht)}. In diesem Beispiel bedeutet das, dass 0, 1, oder mehr Zeichen auf das \enquote{Be...} folgen können.
\end{cheatSheetBox}

%\subsection*{Aggregieren (Zusammenfassen)}
\begin{cheatSheetBox}[title=Aggregatsfunktionen]
Mit Aggregatsfunktionen können Daten zusammengefasst werden, beispielsweise um zu zählen, wie viele Zeilen in einer Tabelle sind (\lstinline|COUNT|), das Maximum / Minimum zu berechnen (\lstinline|MAX| / \lstinline|MIN|), um die Summe oder den Mittelwert einer Spalte zu berechnen (\lstinline|SUM| / \lstinline|AVG|) oder um die Länge eines Texts zu berechnen (\lstinline|LENGTH|). Z.B. berechnet folgender Ausdruck die Anzahl Users in Leipzig:
\begin{lstlisting}
SELECT city, COUNT(*)
FROM users 
WHERE city = 'Leipzig'
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Rechnen und umbenennen: \texorpdfstring{\lstinline|AS|}{AS}]
Mit Spalten sowie Aggregatsfunktionen kann man rechnen, beispielsweise um ein Resultat durch eine andere Zahl zu dividieren. Zudem können Spalten-Titel mit dem Befehl \lstinline|AS| umbenannt werden:
\begin{lstlisting}
SELECT centimeters/100 AS 'Grösse in Metern'
FROM users 
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Gruppieren: \texorpdfstring{\lstinline|GROUP BY|}{GROUP BY}]
Aggregatsfunktionen können auch pro Gruppe verwendet werden, z.B. um die Anzahl Mitglieder in jeder Stadt zu berechnen:
\begin{lstlisting}
SELECT city, COUNT(*) AS 'Users pro Stadt'
FROM users
GROUP BY city
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Filtern \textit{nach} Gruppieren: \texorpdfstring{\lstinline|HAVING|}{HAVING}]
Mit \lstinline|HAVING| können Resultate \textit{nach dem} Gruppieren gefiltert werden. \lstinline|WHERE| filtert \textit{vor} dem Gruppieren und steht demnach immer vor einem \lstinline|GROUP BY|.

Durchschnittliche Körpergrösse aller männlichen Mitglieder in jeder Stadt berechnen, danach auf Städte beschränken, in denen die Menschen durchschnittlich zwischen 150 und 155 gross sind:
\begin{lstlisting}
SELECT city, AVG(centimeters) AS 'Körpergrösse'
FROM users
WHERE gender = 'male'
GROUP BY city
HAVING `Körpergrösse` BETWEEN 150 AND 155
\end{lstlisting}
\end{cheatSheetBox}
Texte werden immer innerhalb von Anführungszeichen (\lstinline|''|) geschrieben. Spaltennamen, welche Spezialzeichen oder Abstände enthalten, werden innerhalb von \textit{backticks} (\lstinline|``|) geschrieben.

\begin{cheatSheetBox}[title=Tabellen erstellen: \texorpdfstring{\lstinline|CREATE TABLE|}{CREATE TABLE}]
Eine Tabelle \texttt{meinetabelle} kann wie folgt erstellt werden:
\begin{lstlisting}
CREATE TABLE meinetabelle(
  name_spalte1 spaltentyp1,
  name_spalte2 spaltentyp2,
  ... --etc.
)

\end{lstlisting}
Gängige Spalten-Typen sind:
\begin{table}[H]
\footnotesize
\centering
\begin{tabular}{l|p{.62\linewidth}}
\lstinline|INTEGER|&Ganzzahl\newline Z.B. 35\\\midrule
\lstinline|REAL| & Kommazahl\newline Z.B. 3.341\\\midrule
\lstinline|DATE|&Datum\newline Format: \lstinline|'YYYY-MM-DD'|\\\midrule
\lstinline|BOOLEAN|& Wahrheitswert\newline Wahr (1) oder Falsch (0)\\\midrule
\lstinline|VARCHAR(n)|&Text \newline\texttt{n} = maximale Länge\\
\end{tabular}
\end{table}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Tabellen ändern: \texorpdfstring{\lstinline|ALTER|}{ALTER}]
Mit dem Befehl \lstinline|ALTER tabellenname| können Tabellen verändert werden, beispielsweise um eine Spalte hinzuzufügen oder umzubenennen.
\begin{lstlisting}
ALTER TABLE tabellenname
ADD neue_spalte eigenschaften
\end{lstlisting}
\begin{lstlisting}
ALTER TABLE tabellenname
RENAME COLUMN name_vorher TO name_neu;
\end{lstlisting}

\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Tabellen löschen: \texorpdfstring{\lstinline|DROP TABLE|}{DROP TABLE}]
\begin{lstlisting}
DROP TABLE tabellenname
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title={Überprüfen, ob existiert: \texorpdfstring{\lstinline|IF EXISTS|}{IF EXISTS}}]
Wenn man eine Tabelle erstellen will, welche es bereits gibt, kann eine Fehlermeldung erscheinen. Folgender Befehl wird nur ausgeführt, falls es noch keine Tabelle \texttt{testtabelle} gibt:
\begin{lstlisting}
CREATE TABLE IF NOT EXISTS testtabelle(
  name_spalte1 spaltentyp1,
  name_spalte2 spaltentyp2,
  ... --etc.
)

\end{lstlisting}
Ähnlich kann vor man den Befehl \lstinline|DROP TABLE| anpassen, so dass er nur ausgeführt wird, wenn es eine solche Tabelle wirklich gibt:
\begin{lstlisting}
DROP TABLE IF EXISTS testtabelle
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Daten einfügen: \texorpdfstring{\lstinline|INSERT INTO|}{INSERT INTO}]
Neue Einträge (Zeilen) können mit dem Befehl \lstinline|INSERT INTO| in eine Tabelle eingefügt werden. Beispielsweise können mit folgender Befehlsstruktur drei neue Zeilen eingefügt werden:
\begin{lstlisting}
INSERT INTO tabellenname 
	(spalte1, spalte2, ...)
VALUES 
	(wert1, wert2, ...),
	(wert1, wert2, ...),
	(wert1, wert2, ...)
\end{lstlisting}
Zuerst werden also die Spalten angegeben, für welche Werte eingefügt werden, danach die Werte für jede neue Zeile. Werte in nicht angegebenen Spalten bleiben leer.
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Daten verändern: \texorpdfstring{\lstinline|UPDATE|}{UPDATE}]
Existierende Daten können wie folgt geändert werden:
\begin{lstlisting}
UPDATE tabelle_name
SET spalte1 = wert1, ...
WHERE bedingung(en)
\end{lstlisting}

Beispielsweise können mit folgendem Befehl alle Users, die zur Stadt Berlin gehören, der Stadt Bern zugeordnet werden:

\begin{lstlisting}
UPDATE users
SET city='Bern'
WHERE city='Berlin'
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Einträge löschen: \texorpdfstring{\lstinline|DELETE FROM|}{DELETE FROM}]
Einzelne Einträge (Zeilen) können mit einer \lstinline|WHERE|-Bedingung und dem Befehl \lstinline|DELETE FROM| gelöscht werden:
\begin{lstlisting}
DELETE FROM users
WHERE username='guenther37'
\end{lstlisting}
\end{cheatSheetBox}

%\subsection*{Tabellenverbindungen}
\begin{cheatSheetBox}[title=Mehrere Tabellen verbinden: \texorpdfstring{\lstinline|JOIN|}{JOIN}]
Mehrere Tabellen können mit folgendem Befehl zu einer Tabelle verbunden werden:

\begin{lstlisting}
SELECT spalte1, spalte2, ...
FROM tabelle1 
JOIN tabelle2 
ON tabelle1.spalte_id1 = tabelle2.spalte_id2
\end{lstlisting}

Falls Primär- und Sekundärschlüssel in beiden Tabellen gleich heissen, kann man \lstinline|USING| verwenden:
\begin{lstlisting}
SELECT spalte1, spalte2, ...
FROM tabelle1 JOIN tabelle2 USING (Spalte_ID)
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Verschachtelte Abfragen]
Verschachtelte \acrshort{SQL}-Abfragen können wie folgt geschrieben werden:
\begin{lstlisting}
SELECT spalte11 
FROM tabelle1
WHERE spaltenname = ...
    (SELECT spalte2 FROM tabelle2 WHERE ...);
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Mehrere Ausdrücke verbinden]
Mehrere SQL-Ausdrücke können mit Semikolon (\texttt{;}) verbunden werden:
\begin{lstlisting}
-- Tabelle erstellen, danach wieder löschen
CREATE TABLE IF NOT EXISTS testtabelle(
  name_spalte1 spaltentyp1,
  name_spalte2 spaltentyp2,
  ... --etc.
);
DROP TABLE IF EXISTS testtabelle
\end{lstlisting}
\end{cheatSheetBox}

\begin{cheatSheetBox}[title=Kommentare]
Kommentare werden von SQL ignoriert und dienen der besseren Leserlichkeit des Codes. Kommentare werden durch zwei Bindestriche gekennzeichnet (s. Box zu \enquote{Mehrere Ausdrücke verbinden})
\end{cheatSheetBox}
