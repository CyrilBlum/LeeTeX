\question {\textbf{Berechnungen}}
\begin{parts}
  \part[1]
  Bestimmen Sie $\text{Nummer}(101011)$.
\begin{solution}
\text{Nummer}(101011) = 43
\end{solution}

  \part[1]
  Bestimmen Sie das binäre Wort $x$, sodass $\text{Nummer}(x) = 41$.
\begin{solution}
  Das gesuchte binäre Wort ist $x = 101001$, denn
  \begin{align*}
  \text{Nummer}(101001) = 41.
  \end{align*}
\end{solution}

  \part[1]
  Berechnen Sie (stellen Sie ohne Logarithmus dar) die Zahl $\ceil{\log_3\lr{37}}$.
\begin{solution}
  Es gilt
  \begin{align*}
    \underbrace{3^3}_{= 27} < 37 < \underbrace{3^4}_{= 81}
  \end{align*}
  und somit $\ceil{\log_3\lr{37}} = 4$.
\end{solution}

  \part[1]
  Berechnen Sie $\abs{\text{Bin}(1078586)}$. 

  Tipp:
  \begin{align*}
    2^{19} = 524288, \quad 2^{20} = 1048576, \quad 2^{21} = 2097152
  \end{align*}
\begin{solution}
Es gilt
\begin{align*}
  \abs{\text{Bin}(1078586)} = \ceil{\log_2\lr{1078586 + 1}}.
\end{align*}
Wegen
\begin{align*}
 2^{20} < 1078586 + 1 < 2^{21} \iff 20 < \log_2(1078586 + 1) < 21
\end{align*}
folgt
\begin{align*}
  \abs{\text{Bin}(1078586)} = \ceil{\log_2\lr{1078586 + 1}} = 21.
\end{align*}
\end{solution}

  \part[1]
  Sei $y$ die binäre Darstellung ($2$-adische Darstellung) der zweitgrössten Zahl, welche mit $11$ Bits dargestellt werden kann. 
  \begin{subparts}
    \subpart[1] Wie sieht $y$ aus?
\begin{solution}
  $y = 11111111110$
\end{solution}
    \subpart[1] Bestimmen Sie $\text{Nummer}\lr{y}$.
\begin{solution}
  Die grösste binäre Zahl mit $11$ Stellen ist $z := 11111111111$. Die kleinste binäre Zahl mit $12$ Stellen ist $100000000000 = 2^{11}$ und diese ist um $1$ grösser als $z$. Somit folgt, dass $\text{Nummer}(y) = 2^{11} - 2$. 
\end{solution}
  \end{subparts}
\end{parts}

\question{\textbf{Primzahlen}}

\begin{parts}
  \part[2]
Formulieren Sie den Primzahlsatz.
\begin{solution}
  \begin{align*}
    \lim_{n\to\infty} \frac{\text{Prim}(n)}{n/\ln (n)} = 1.
  \end{align*}
\end{solution}
  \part[1] Eine Primzahllücke (prime gap) ist der Abstand zwischen zwei direkt aufeinanderfolgenden Primzahlen. Die ersten 5 Primzahllücken sind:
  \begin{align*}
    1, 2, 2, 4, 2.
  \end{align*}
  Erklären Sie, warum alle Primzahllücken (ausser der allerersten) gerade sind.
  \begin{solution}
    Die alle Primzahlen abgesehen von der $2$ sind ungerade. Sei $p \neq 2$ eine Primzahl. Angenommen es gäbe eine ungerade Primzahllücke (ausser der ersten) $g$, dann wäre $p + g$ als Summe zweier ungeraden Zahlen eine gerade Zahl und somit sicherlich keine Primzahl. Dann kann $g$ aber keine Primzahllücke sein.
  \end{solution}
\end{parts}

\question[2] Im Unterricht haben wir die Python \pythoninline{sieb} wie in \cref{listing:sieb} implementiert. 
Schreiben Sie eine Python-Funktion \pythoninline{def Prim(n)}, welche \pythoninline{sieb} verwendet um die Funktion $\text{Prim}(n)$ für eine gegebene natürliche Zahl $n$ zu berechnen.

\begin{lstlisting}[language=Python,caption=Sieb des Eratosthenes,numbers=none,label=listing:sieb]
def sieb(n):
    primzahlen = []
    nicht_gestrichen = (n + 1) * [True]

    i = 2
    while i <= n:
        if nicht_gestrichen[i]:
            primzahlen.append(i)
            j = 2 * i
            while j <= n:
                nicht_gestrichen[j] = False
                j += i
        i += 1
    return primzahlen
\end{lstlisting}

\begin{lstlisting}[numbers=none]
  Testfälle:

  Prim(4)  sollte printen: 2
  Prim(20) sollte printen: 8
  Prim(21) sollte printen: 8
\end{lstlisting}
\begin{solution}
\begin{lstlisting}[language=Python,caption=Prim,numbers=none]
def prim(n):
    return len(sieb(n))
\end{lstlisting}
\end{solution}

\question[3]

Ein Kommunikationsprotokoll hat eine Fehlerwahrscheinlichkeit von $r\in\io{0}{1}$. Wie oft muss das Protokoll \underline{mindestens} wiederholt werden, bis die gesamte Fehlerwahrscheinlichkeit (nach wiederholter Durchführung des Protokolls) kleiner als $w$ ist. Dabei gilt $w\in\io{0}{1}$. 
\begin{solution}
  Wir suchen die kleinste natürliche Zahl (Anzahl) $n$, sodass
  \begin{align*}
    r^n < w.
  \end{align*}
  Auslösen der Ungleichung gibt uns
  \begin{align*}
    n = \ceil{\frac{\ln(w)}{\ln(r)}}.
  \end{align*}
\end{solution}

\question[3] Gegeben sind die beiden Datensätze
\begin{itemize}
  \item $x := 000000$ in Zürich und
  \item $y := 101010$ in Boston.
\end{itemize}
Berechnen Sie die exakte Wahrscheinlichkeit, dass das Kommunikationsprotokoll in diesem Fall falsch entscheidet.
\begin{solution}
  Die beiden Datensätze $x$ und $y$ haben jeweils eine Länge von $n := 6$ Bits.
  Wir berechnen die Grösse
  \begin{align*}
    w := \abs{\text{Nummer}(x) - \text{Nummer}(y)} = 42.
  \end{align*}
Die Zahl $42$ besitzt die Primfaktorisierung $42 = 2\cdot 3\cdot 7$. Es existieren also genau drei Primzahlen, welche die Differenz $w$ teilen. Insgesamt gibt es aber $11$ Primzahlen, die kleiner als $n^2 = 36$ sind (und gewählt werden könnten). Damit beträgt die Fehlerwahrscheinlichkeit $3/11$.  
\end{solution}
