\subsection{Einfache Selektion: \lstinline[style=sql]|SELECT|}

Mit dem Befehl ``\lstinline[style=sql]|SELECT col1,col2,... FROM tablename|'' können wir gewisse Spalten (=Kolonnen) \lstinline[style=sql]|col1,col2,...| aus einer Tabelle mit dem Namen \lstinline[style=sql]|tablename| auswählen. Wir können statt den Spaltennamen auch einfach einen Stern (\texttt{*}) tippen, um alle Kolonnen einer Tabelle zu erhalten, d.h. die gesamte Tabelle.

\textbf{Aufgabe} Geben Sie auf InstaHub folgenden Befehl ein:
\begin{lstlisting}[style=sql]
SELECT *
FROM users
\end{lstlisting}

Mit diesem Befehl wählen wir alle Spalten (\lstinline[style=sql]|*|) der Tabelle \lstinline[style=sql]|users| aus. 
Sie sollten nun die gesamte Tabelle \texttt{users} sehen.

\textbf{Aufgaben}: 
\begin{itemize}
\item Geben Sie alle Benutzernamen (\texttt{username}) aus \texttt{users} aus.
\item Geben Sie alle Benutzernamen (\texttt{username}) und echten Namen (\texttt{name}) aller Einträge aus \texttt{users} aus.
\item Geben Sie die Namen und Wohnorte aller Mitglieder aus.
\end{itemize}

\subsection{Selektion mit Filter: \lstinline[style=sql]|WHERE|}
Fügen wir zu einer SQL-Abfrage den Befehl ``\lstinline[style=sql]|WHERE conditions|'' hinzu, können wir können wir das erhaltene Resultat nach gewissen Bedingungen (\texttt{conditions}) filtern. 

\textbf{Aufgabe}: Geben Sie folgendes Beispiel in Instahub ein:

\begin{lstlisting}[style=sql]
SELECT name, city, gender
FROM users
WHERE gender="male"
\end{lstlisting}

Nebst dem Gleich-Zeichen (im obigen Beispiel) haben folgende Filter-Operatoren zur Verfügung:
\begin{table}[H]
 \centering
 \begin{tabular}{c|l}
 \makecell{\textbf{Operator}\\\textbf{in SQL}} & \makecell{\textbf{Mathematische}\\\textbf{Bedeutung}} \\\midrule 
      \texttt{=} & gleich ($=$)\\
      \texttt{<>} & ungleich ($\neq$) \\
      \texttt{<} & kleiner als ($<$)\\
      \texttt{<=} & kleiner oder gleich ($\leq$) \\
      \texttt{>} & grösser als ($>$) \\
      \texttt{>=} & grösser oder gleich ($\geq$)
 \end{tabular}
\end{table}
Um mehrere Bedingungen miteinander zu verknüpfen, können wir die Befehle \lstinline[style=sql]|AND|, \lstinline[style=sql]|AND NOT| sowie \lstinline[style=sql]|OR| verwenden.

\textbf{Aufgaben}:
\begin{itemize}
\item Geben Sie alle Mitglieder aus, die über 1,85 Meter gross und männlich sind.
\item \textbf{Challenge}: Überprüfen Sie, ob es minderjährige Mitglieder gibt (mit der Spalte \texttt{birthday}).
\end{itemize}

\subsection{Eindeutige Selektion: \lstinline[style=sql]|SELECT DISTINCT|}

Der Befehl \lstinline[style=sql]|DISTINCT| nach einem \lstinline[style=sql]|SELECT| führt dazu, dass jeder mögliche Wert nur einmal ausgegeben wird, d.h. Duplikate werden entfernt. Vergleichen Sie folgende zwei Befehle:

\begin{lstlisting}[style=sql]
SELECT gender
FROM users
\end{lstlisting}

\begin{lstlisting}[style=sql]
SELECT DISTINCT gender
FROM users
\end{lstlisting}

\textbf{Aufgaben}:
\begin{itemize}
\item Geben Sie jeden Wohnort nur einmal aus.
\item Geben Sie jede Benutzerrolle nur einmal aus. \texttt{dba} bedeutet ``Database Administrator'', also Datenbank-AdministratorIn.
\end{itemize}


\subsection{Ungefähre Matches: \lstinline[style=sql]|LIKE|}

Mit dem Befehl \lstinline[style=sql]|LIKE| können wir die Tabelle nach Spalten zu filtern, welche ein bestimmtes Wort enthalten. Beispielsweise erhalten wir mit folgendem Befehl alle Städte, die mit ``Be...'' beginnen:

\begin{lstlisting}[style=sql]
SELECT DISTINCT city 
FROM users 
WHERE city LIKE "Be%"
\end{lstlisting}

Das ``\%''-Zeichen dient hier als Platzhalter und bedeutet ``es kann noch weiterer Text an dieser Stelle stehen, muss aber nicht''.

\textbf{Aufgabe}:  Geben Sie folgende Befehle in InstaHub ein und beobachten Sie das Resultat:

\begin{lstlisting}[style=sql]
SELECT username, city 
FROM users 
WHERE city = "Berlin" AND name LIKE "Fabian%"
\end{lstlisting}

\begin{lstlisting}[style=sql]
SELECT username, city 
FROM users 
WHERE city = "Berlin" OR name LIKE "Fabian%"
\end{lstlisting}

\begin{lstlisting}[style=sql]
SELECT username, city 
FROM users 
WHERE city = "Berlin" AND NOT name LIKE "Fabian%"
\end{lstlisting}

Aufgaben:
\begin{itemize}
\item Finden Sie alle Berliner die Marc heissen.
\item Finden Sie alle Linas und Lorenas.
\item Zeigen Sie das Geburtsdatum und den Benutzernamen aller Frauen an, die kleiner als 1,60 Meter sind.
\item \emoji{up-arrow} \textcolor{red}{\textbf{Abgabe}}: Wählen Sie alle Felix aus, die nicht aus Berlin kommen.
\item \textbf{Challenge}: Sortieren Sie alle Männer, welche mindestens 16-jährig sind, nach ihrer Körpergrösse.
\end{itemize}

\subsection{Sortieren: \lstinline[style=sql]|ORDER BY|}

Mit dem Befehl \lstinline[style=sql]|ORDER BY col ASC| oder \lstinline[style=sql]|ORDER BY col DESC| kann das Resultat einer SQL-Abfrage nach einer bestimmten Kolonne \lstinline[style=sql]|col| sortiert werden. Mit \lstinline[style=sql]|ASC| wird das Resultat aufsteigend und mit \lstinline[style=sql]|DESC| absteigend sortiert.

\textbf{Aufgabe}: Beobachten Sie das Resultat folgender Abfrage:

\begin{lstlisting}[style=sql]
SELECT DISTINCT city 
FROM users 
ORDER BY city DESC
\end{lstlisting}

\subsection{Erste \textit{n} Zeilen: \lstinline[style=sql]|LIMIT|}

Mit dem Befehl \lstinline[style=sql]|LIMIT| kann das Resultat einer SQL-Abfrage auf eine gewisse Anzahl Zeilen beschränkt werden.

\textbf{Aufgabe}: Beobachten Sie das Resultat folgender Abfrage:

\begin{lstlisting}[style=sql]
SELECT DISTINCT city
FROM users
LIMIT 5
\end{lstlisting}

\begin{itemize}
\item Zeigen Sie nur 3 Mitglieder an.
\item Zeigen Sie nur die 5 grössten Mitglieder an.
\end{itemize}

\subsection{Zählen: \lstinline[style=sql]|COUNT|}

Mit dem Befehl \lstinline[style=sql]|COUNT| kann man die Anzahl Resultate zählen:


\begin{lstlisting}[style=sql]
SELECT COUNT(*) AS "Registrierte Mitglieder" 
FROM users
\end{lstlisting}
Ermitteln Sie mithilfe des obigen Befehls, wie viele Mitglieder dein InstaHub hat.


\subsection{\lstinline[style=sql]|MIN|/\lstinline[style=sql]|MAX|}

Um die höchsten, bzw. tiefsten Werte einer Kolonne zu erhalten, können die Befehle \lstinline[style=sql]|MIN| und \lstinline[style=sql]|MAX| verwendet werden:

\textbf{Aufgabe}: Beobachten Sie das Resultat folgender Abfrage:

\begin{lstlisting}[style=sql]
SELECT MAX(centimeters) AS "Maximale Körpergrösse" 
FROM users
\end{lstlisting}

\begin{itemize}
\item Zeigen Sie, wie gross das kleinste Mitglied ist.
\item Zeigen Sie, wie alt als das älteste Mitglied ist.
\item Zeigen Sie, wann sich zuletzt ein Mitglied registriert hat.
\end{itemize}

\subsection{Gruppieren: \lstinline[style=sql]|MIN|/\lstinline[style=sql]|GROUP BY|}
Um die Resultate einer Anfrage pro Untergruppen zu sehen, kann der Befehl \lstinline[style=sql]|GROUP BY| verwendet werden:
\begin{lstlisting}[style=sql]
SELECT city, COUNT(*) AS "Mitglieder je Stadt" 
FROM users 
GROUP BY city
\end{lstlisting}
\emoji{up-arrow} \textcolor{red}{\textbf{Abgabe}}: Geben Sie das grösste Mitglied in jeder Stadt aus
