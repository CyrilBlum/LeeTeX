In diesem Skript lernen Sie einige zentralen Algorithmen sowie Verfahren des überwachten Maschinellen Lernens kennen.

\begin{mydefinition}[Algorithmus]
Ein \textbf{Algorithmus} ist eine eindeutige, endliche Abfolge von Anweisungen, die ein Problem lösen oder eine Aufgabe erfüllen. Er nimmt Eingabedaten, verarbeitet sie nach definierten Regeln und liefert eine Ausgabe.
\end{mydefinition}

\begin{myexample}
\begin{itemize}
    \item Ein Kochrezept ist ein Algorithmus, der Zutaten (Eingabe) in ein Gericht (Ausgabe) verwandelt.
    \item Ein Navigationssystem verwendet Algorithmen, um die beste Route von A nach B zu berechnen.
    \item Ein Spamfilter ist ein Algorithmus, der E-Mails (Eingabe) als \enquote{Spam} oder \enquote{kein Spam} klassifiziert (Ausgabe).
\end{itemize}
\end{myexample}

\begin{mydefinition}[\acrfull{ML}]
Maschinelles Lernen (\gls{ML}) ist ein Teilgebiet der \glslink{KI}{künstlichen Intelligenz}.
Anstatt explizite Regeln zu programmieren, \textbf{lernt} ein \gls{ML}-Modell
Muster direkt aus Daten.
\end{mydefinition}

\begin{myexample}[Beispiele von \gls{ML}]
    Einige Beispiele, wie \gls{ML} im Alltag eingesetzt wird:
\begin{itemize}
    \item Statt zu programmieren \enquote{Wenn Wort \texttt{Gewinn} vorkommt, dann Spam},
          lernt ein Modell selbst aus Tausenden von Beispiel-E-Mails.
    \item Statt Regeln für Handschrift, lernt ein Modell aus Millionen beschrifteter Ziffernbilder.
\end{itemize}
\end{myexample}

\section{Klassifikation vs. Regression}

Im maschinellen Lernen gibt es verschiedene Arten von Aufgaben. Zwei der wichtigsten sind
\textbf{Klassifikation} und \textbf{Regression}. Sie unterscheiden sich darin, was genau vorhergesagt wird: während die Klassifikation Kategorien vorhersagt, sagt die Regression kontinuierliche Werte (Zahlen) vorher.

\begin{mydefinition}[Klassifikation]
Bei der \textbf{Klassifikation} wird ein Datenpunkt einer
\textbf{Kategorie} zugeordnet, z.\,B. \enquote{bestanden / nicht bestanden} oder
\enquote{Spam / kein Spam}.
\end{mydefinition}

\begin{mydefinition}[Regression]
Bei der \textbf{Regression} wird ein \textbf{kontinuierlicher Zahlenwert}
vorhergesagt, z.\,B. eine Wohnungsmiete oder der Stromverbrauch nächsten Monat.
\end{mydefinition}

\begin{myexample}
    Folgende Aufgaben sind Beispiele für Klassifikation und Regression:
\begin{itemize}
    \item \textbf{Klassifikation:} Ist eine E-Mail Spam oder kein Spam?
    \item \textbf{Regression:} Wie hoch wird die Miete für eine 80-m\textsuperscript{2}-Wohnung sein?
\end{itemize}
\end{myexample}

\begin{myexercise}[Warm-up: Problemtyp erkennen]
Ordnen Sie zu, ob es sich um \textbf{Klassifikation} oder \textbf{Regression} handelt:
\begin{enumerate}
    \item Vorhersage der Aussentemperatur morgen um 14:00 Uhr
    \item Erkennung, ob ein Bild eine Katze oder einen Hund zeigt
    \item Schätzung des Wiederverkaufspreises eines Fahrrads
    \item Erkennung, ob eine Transaktion betrügerisch ist
    \item Prognose der Anzahl verkaufter Tickets für ein Konzert
    \item Einordnung einer Filmkritik als positiv oder negativ
\end{enumerate}
\begin{myanswer}
Regression: 1, 3, 5 \hspace{1cm} Klassifikation: 2, 4, 6
\end{myanswer}
\end{myexercise}

\textbf{Filterung} kann sowohl als Klassifikation (z.\,B. Spamfilter) als auch als Regression (z.\,B. Content-Ranking) auftreten -- die Grenzen sind fliessend. 

\textbf{Assoziation} ist eine eigene Kategorie, da sie nicht direkt Vorhersagen trifft, sondern Muster in den Daten entdeckt (z.\,B. \enquote{Kunden kauften auch...}). Dies kann sowohl regelbasiert als auch selbstlernend erfolgen und sowohl für Klassifikation als auch für Regression relevant sein.

\textbf{Priorisierung} ist ebenfalls eine eigene Kategorie, da sie nicht nur klassifiziert oder regrediert, sondern auch eine Rangfolge erstellt (z.\,B. Google-Suchergebnisse). Dies kann sowohl regelbasiert als auch selbstlernend erfolgen und ist oft mit Klassifikation oder Regression kombiniert, aber die Hauptfunktion ist die Erstellung einer Prioritätenliste.

Statt einer strikten Einteilung in Kategorien, ist es oft hilfreicher, Algorithmen entlang von zwei Dimensionen zu betrachten: \textbf{Funktionsart}, also was getan wird (z.\,B. Priorisierung, Klassifikation, Assoziation, Filterung) und \textbf{Umsetzungsart} (regelbasiert vs. selbstlernend). Dies ermöglicht eine flexiblere Einordnung und zeigt, dass viele Algorithmen mehrere Funktionen erfüllen können.

\section{Gesellschaftliche Aspekte: Verantwortungsvolles Machine Learning}

\begin{mydefinition}[Responsible AI]
Je mehr Entscheidungen durch automatisierte Systeme getroffen werden, desto wichtiger werden Fragen nach Fairness, Transparenz und Rechenschaftspflicht. \textbf{Responsible AI} bedeutet, ML-Systeme so zu entwickeln, dass sie:
\begin{itemize}
    \item Die Auswirkungen auf betroffene Personen berücksichtigen
    \item Bias und Diskriminierung vermeiden oder zumindest dokumentieren
    \item Erklärbar sind (oder ihre Grenzen klar kommunizieren)
    \item Regulatorische Anforderungen erfüllen
\end{itemize}
\end{mydefinition}

\begin{myexample}[Auswirkungen von Modellen im echten Leben]
Die Algorithmen in diesem Skript sind nicht nur Schulübungen. Sie werden täglich verwendet für:
\begin{itemize}
    \item \textbf{Kreditvergabe}: Entscheidungsbaum oder Random Forest bestimmen, wer einen Kredit erhält
    \item \textbf{Stellenbesetzung}: Naive Bayes und KNN filtern Bewerbungen
    \item \textbf{Medizinische Diagnose}: Klassifikatoren helfen, Krankheiten zu erkennen (oder nicht)
    \item \textbf{Kriminaljustiz}: Prognose-Algorithmen beeinflussen Haftentscheidungen
\end{itemize}
Jede falsche Vorhersage kann eine echte Person negativ beeinflussen.
\end{myexample}

\begin{mychallenge}[Reflexion: Verantwortung und Abwägungen]
Stellen Sie sich vor, Sie entwickeln einen Algorithmus für eine Krankenkasse, welche damit Versicherungskosten bestimmt.
\begin{enumerate}
    \item Wem sollte Sie gegenüber verantwortlich sein? Der Versicherung? Den Kundinnen? Der Öffentlichkeit?
    \item Welche Art von Fehler ist schlimmer: False Positive (Mensch wird zu teuer eingestuft) oder False Negative (Risiko wird übersehen)?
    \item Wie würden Sie testen, ob Ihr Modell Geschlecht oder ethnische Herkunft diskriminiert -- auch wenn diese Features nicht explizit im Datensatz sind?
\end{enumerate}
\begin{myanswer}
    \begin{enumerate}
        \item Idealerweise gegenüber allen: aber Interessenskonflikte sind real. Transparenz über Trade-offs ist wichtig.
        \item Je nach Kontext. Versicherung könnte False Positives bevorzugen (höhere Prämien, aber weniger Ausfallrisiko). Menschen bevorzugen False Negatives (günstigere Prämien). Eine \enquote{neutrale} Entscheidung existiert nicht.
        \item Durch Bias-Audits: Vergleiche Fehlerquoten zwischen Gruppen, prüfe auf Proxy-Variablen (z.~B. Wohnort als Proxy für Ethnizität), teste auf Performance-Unterschiede.
    \end{enumerate}
\end{myanswer}
\end{mychallenge}
