\section{Listen}
\subsection{Einführung in Listen}
Bisher haben wir uns mit Variablen beschäftigt, die nur einen Wert speichern können::
\begin{itemize}
	\item \lstinline|x = 3| (eine Zahl / \enquote{integer})
	\item \lstinline|name = "Hallo"| (einen Text /  \enquote{string})
	\item \lstinline|ist_wahr = True| (einen Wahrheitswert / \enquote{boolean})
\end{itemize}

In der Informatik ist es jedoch oft nötig, mit vielen Werten gleichzeitig arbeiten zu können, gerade im Kontext von \textit{Big Data}. Eine Möglichkeit, in Python mit vielen Werten zu arbeiten, sind \textbf{Listen}.

\begin{mydefinition}
Eine \textbf{Liste} ist eine Sammlung von Werten, die in einer Variablen gespeichert werden können. Eine Liste kann beliebig viele Werte enthalten, und diese Werte können von einem beliebigen Typ \enquote{integer}, \enquote{string} oder \enquote{boolean} sein.
\end{mydefinition}

\begin{myexample}[Listen erstellen]
Folgendes Beispiel zeigt, wie eine Liste in Python definiert wird. Wir können dabei, wie auch bei anderen Variablentypen, beliebige Namen verwenden. Die Inhalte der Liste werden in eckigen Klammern \lstinline|[]| geschrieben und die einzelnen Werte werden durch Kommas \lstinline|,| getrennt.
\begin{lstlisting}
# Liste mit Zahlen  
l1 = [1, 2, 3, 4, 5]
# Liste mit Texten
l2 = ["Hallo", "Welt", "Python"]
# Liste mit Wahrheitswerten
l3 = [True, False, True]
# Liste mit gemischten Werten
l4 = [1, "Hallo", True, 3.14]
\end{lstlisting}
\end{myexample}
\begin{myexample}[Zugriff auf Listen]
Wenn wir auf die einzelnen Werte in der Liste zugreifen möchten, können wir dies mit dem Index tun. Der Index ist eine Zahl, die angibt, an welcher Stelle sich der Wert in der Liste befindet. Der Index beginnt bei 0, das heisst, der erste Wert in der Liste hat den Index 0, der zweite Wert hat den Index 1 und so weiter.
\begin{lstlisting}
# Liste l1 erstellen
l1 = [1, 2, 3, 4, 5]
# Zugriff auf das erste Element der Liste
print(l1[0]) # gibt 1 aus
# Zugriff auf das zweite Element der Liste
print(l1[1]) # gibt 2 aus
# Zugriff auf das letzte Element der Liste
print(l1[-1])
\end{lstlisting}
\end{myexample}
\begin{myexample}[Listen-Werte verändern]
Um den Wert an einer bestimmten Stelle in der Liste zu ändern, können wir ebenfalls den Index verwenden. Wir können den Wert an dieser Stelle einfach durch einen neuen Wert ersetzen.
\begin{lstlisting}
# Liste l1 erstellen
l1 = [1, 2, 3, 4, 5]
# Ändern des Wertes an der Stelle 0 in der Liste l1
l1[0] = 10
print(l1) # gibt [10, 2, 3, 4, 5] aus
\end{lstlisting}
\end{myexample}
\begin{myexample}[Länge einer Liste]
Der Befehl \lstinline|len(liste)| gibt die Länge der Liste zurück, also die Anzahl der Werte, die in der Liste gespeichert sind. Dies ist nützlich, wenn wir wissen möchten, wie viele Werte in der Liste enthalten sind.
\end{myexample}

\subsubsection{Sortier-Algorithmen}
Eine der häufigsten Anwendungen in der Informatik ist das Sortieren von Daten. Es gibt viele verschiedene Algorithmen, um Daten zu sortieren, und jeder Algorithmus hat seine eigenen Vor- und Nachteile, insbesondere hinsichtlich der Geschwindigkeit und der benötigten Rechenleistung. Im Folgenden wird der Bubble-Sort-Algorithmus vorgestellt, der eine einfache Methode ist, um Daten zu sortieren. Der Bubble-Sort-Algorithmus funktioniert, indem er die Liste von Werten durchläuft und benachbarte Werte vergleicht. Wenn ein Wert grösser ist als der nächste Wert, werden die beiden Werte vertauscht. Dieser Vorgang wird so lange wiederholt, bis die gesamte Liste sortiert ist. Die ersten zwölf Schritte des Bubble-Sort-Algorithmus sind in \cref{fig:bubble-sort} dargestellt. Der Algorithmus wird so lange wiederholt, bis die gesamte Liste sortiert ist.

Der Algorithmus hat eine Zeitkomplexität von $O(n^2)$, was bedeutet, dass die Laufzeit des Algorithmus quadratisch mit der Anzahl der Werte in der Liste wächst. Dies macht den Bubble-Sort-Algorithmus für grosse Datenmengen ineffizient. Aufgrund der inneren Schleife (\lstinline|for _ in range(n - 1 - i)|) kann die Laufzeit noch geringfügig verbessert werden: In jedem Durchlauf der äusseren Schleife ist das jeweils grösste Element bereits an die richtige Position gewandert, sodass die innere Schleife jedes Mal um ein Element kürzer wird.

\begin{itemize}
	\item \textbf{nicht optimierte Version:} Die Anzahl der Vergleiche beträgt \[(n-1)^2\]
	      Das bedeutet: Für eine Liste mit 6 Elementen sind das $5^2 = 25$ Vergleiche.
	\item \textbf{optimierte Version:} Die Anzahl der Vergleiche entspricht der Summe der Zahlen von 1 bis $n-1$:
	      \[
		      (n-1) + (n-2) + \ldots + 2 + 1 = \displaystyle\sum_{m=1}^{n-1} m.
	      \]
	      Für eine Liste mit 6 Elementen ergibt das $\displaystyle\sum_{m=1}^{5} m = 1 + 2 + 3 + 4 + 5 = 15$ Vergleiche.
	\item Im Mathematikunterricht lernen Sie, dass die Gleichheit
	      \[
		      (n-1) + (n-2) + \ldots + 2 + 1 = \displaystyle\sum_{m=1}^{n-1} m = \textcolor{ForestGreen}{\frac{1}{2}n^2 + \frac{1}{2}n}
	      \]
	      gilt. Damit bleibt die Anzahl der Vergleiche von Bubble-Sort auch mit dieser Optimierung im Wesentlichen quadratisch.
\end{itemize}

Im Allgemeinen wächst die Anzahl der Vergleiche beim Bubble-Sort-Algorithmus quadratisch mit der Länge der Liste, also $O(n^2)$. Die Optimierung reduziert die Anzahl der Vergleiche, ändert aber nichts an der grundsätzlichen Komplexität.

\subsubsection{Such-Algorithmen}
In der Informatik stellt sich häufig die Frage, wie man möglichst schnell herausfinden kann, ob eine bestimmte Zahl in einer Liste enthalten ist.

Ist die Liste unsortiert, wie zum Beispiel bei \lstinline|x = [5, 3, 8, 20, 2, 10]|, bleibt uns nichts anderes übrig, als jedes Element der Liste einzeln zu überprüfen. Dies entspricht einer linearen Suche, bei der im ungünstigsten Fall alle Elemente betrachtet werden müssen.

Ist die Liste jedoch bereits sortiert, wie zum Beispiel bei \lstinline|x = [2, 3, 5, 8, 10, 20]|, können wir effizientere Suchverfahren anwenden. Dies lässt sich mit der Suche nach einem Gegenstand in einem Koffer vergleichen. Wenn der Koffer unordentlich gepackt ist, müssen wir jeden einzelnen Gegenstand herausnehmen, um den gesuchten zu finden. Wenn der Koffer jedoch ordentlich gepackt ist, können wir die Gegenstände viel schneller finden (s. \cref{fig:koffer-suche}).

\subsection{Listen verändern}
\subsubsection{Elemente zu Listen hinzufügen und entfernen}

In Python gibt es verschiedene Methoden, um Listen dynamisch zu verändern. Die wichtigsten Methoden sind \lstinline|append|, \lstinline|insert| und \lstinline|pop|.

\begin{mydefinition}[Listen verändern]
Listen könnnen in Python dynamisch verändert werden, indem Elemente hinzugefügt oder entfernt werden. Die wichtigsten Methoden dafür sind:
\begin{itemize}
	\item \lstinline|liste.append(wert)| fügt \textbf{am Ende} der Liste einen neuen Wert hinzu.
	\item \lstinline|liste.insert(index, wert)| fügt an der angegebenen Position (\texttt{index}) einen neuen Wert ein. Alle nachfolgenden Elemente werden nach rechts verschoben.
	\item \lstinline|liste.pop(index)| entfernt das Element an der angegebenen Position (\texttt{index}) und gibt es zurück. Wird kein Index angegeben, wird das letzte Element entfernt.
\end{itemize}
\end{mydefinition}

\section{Dictionaries}
\subsection{Dictionaries}
In der Informatik werden Daten oft nicht nur als Listen, sondern auch als sogenannte \textbf{Dictionaries} (Wörterbücher) gespeichert. Ein Dictionary ist eine Sammlung von Schlüssel-Wert-Paaren. Anders als bei Listen werden die Werte nicht durch einen Index, sondern durch einen eindeutigen \textbf{Schlüssel} (key) angesprochen. Dies ist vielfach praktischer als die Speicherung in Listen, da man direkt mit einem Begriff auf die Werte zugreifen kann, ohne die Position im Array kennen zu müssen.

\begin{mydefinition}
Ein \textbf{Dictionary} ist eine Datenstruktur, die jedem Schlüssel (\textit{key}) einen Wert (\textit{value}) zuordnet. Die Schlüssel müssen eindeutig sein und können z.B. Zahlen oder Zeichenketten sein.
\end{mydefinition}

\begin{myexample}[Dictionary für Kontaktdaten]
Ein typisches Beispiel für ein Dictionary ist ein Adressbuch, in dem zu jedem Namen die Telefonnummer gespeichert ist:

\begin{lstlisting}
telefonbuch = {
    "Anna": "079 123 45 67",
    "Ben": "078 987 65 43",
    "Clara": "077 555 44 33"
}
print(telefonbuch["Anna"])  # gibt "079 123 45 67" aus
\end{lstlisting}
\end{myexample}

\begin{myexample}[Dictionary für Produktpreise]
Auch in einem Online-Shop werden Produkte oft mit ihren Preisen als Dictionary gespeichert:
\begin{lstlisting}
preise = {
    "Apfel": 0.80,
    "Banane": 0.50,
    "Brot": 2.50
}
print(preise["Brot"])  # gibt 2.5 aus
\end{lstlisting}
\end{myexample}

\begin{myexample}[Werte hinzufügen und ändern]
Sie können einem Dictionary neue Schlüssel-Wert-Paare hinzufügen oder bestehende Werte ändern:
\begin{lstlisting}
preise["Milch"] = 1.60         # neues Produkt hinzufügen
preise["Apfel"] = 0.90         # Preis ändern
print(preise)
\end{lstlisting}
\end{myexample}

\begin{myexample}[Alle Schlüssel und Werte durchgehen]
Mit einer Schleife können Sie alle Einträge eines Dictionaries durchgehen:
\begin{lstlisting}
for produkt in preise:
    print(produkt, "kostet", preise[produkt], "Franken")
\end{lstlisting}
\end{myexample}
